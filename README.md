## Design Patterns (PHP)

## Creational (Порождающие)

Паттерны, отвечающие за создание объектов.

1. **Singleton** — один экземпляр объекта.
2. **Factory** — класс создаёт другой класс.
3. **Factory Method**
4. **Static Factory**
5. **Abstract Factory**
6. **Builder** — пошаговое создание сложного объекта.
7. **Prototype** — клонирование объекта вместо `new`.
8. **Pool** — переиспользование объектов.

---

## Structural (Структурные)

Паттерны, уменьшающие связанность системы.

9. **Dependency Injection** — внедрение зависимостей извне.
10. **Registry** — глобальное хранилище объектов.
11. **Composite** — объект из составных частей.
12. **Adapter** — совместимость разных интерфейсов.
13. **Bridge** — разделение абстракции и реализации.
14. **Data Mapper** — отделение модели от БД.
15. **Decorator** — динамическое расширение поведения.
16. **Facade** — простой интерфейс к сложной системе.
17. **Fluent Interface** — цепочка вызовов (`return $this`).
18. **Flyweight** — экономия памяти.
19. **Proxy** — посредник при доступе к объекту.

---

## Behavioral (Поведенческие)

Паттерны взаимодействия объектов.

20. **State** — изменение поведения через состояние.
21. **Strategy** — выбор алгоритма во время выполнения.
22. **Null Object**
23. **Command** — инкапсуляция запроса в объект.
24. **Interpreter** — реализация языка/DSL.
25. **Specification** — бизнес-правила как объекты.
26. **Chain of Responsibility** — цепочка обработчиков.
27. **Iterator** — последовательный доступ к элементам.
28. **Mediator** — уменьшение связности объектов.
29. **Memento** — сохранение состояния объекта.
30. **Observer** — подписка на события.
31. **Template Method** — каркас алгоритма.
32. **Visitor** — операции над структурами объектов.
