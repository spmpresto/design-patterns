Creational [Порождающие]:

1. Singleton
2. Factory [класс создает другой класс]
3. Factory Method
4. Static Factory
5. Abstract Factory
6. Builder [класс который оперирует другими классами и дает какой то результат (как пример сообщение для отправки, которое состоит из нескольких частей, за которую отвечает отдельный класс)]
7. Prototype [чтобы не использовать слово new, используется слово clone]
8. Pool [когда объектов в массиве нет он создаст объект]


Structural [Структурные (снизить монолитность приложения, создаем один класс который будет связующим звеном с другими классами)]:

9. Dependency Injection (внедрение зависимостей) [хотим создать объект(класс) , но всю конфигурацию объекта помещаем в отдельный класс. конфигурация хранится в одном классе, а с другого класса получаем всю конфигурацию, при этом не меняя класс, где хранится конфигурация]
10. Registry (Реестр) (это как хранилище объектов)
11. Composite (Компоновщик) [из составных частей собирается что то целое (очень похож на шаблон builder)]
12. Adapter [Адаптер является связующим звеном между двумя классами. Есть реализация двух классов, которые схожи, но есть некоторые отличия в реализации и при запуске одного класса надо вызывать схожий метод у второго класса ]
13. Bridge (Мост) [похож на Adapter]
14. Data Mapper [хранится массив из объектов, где по ключу можно получить объект]
15. Decorator [класс который является точкой, в которой можно запускать методы другого класса]
16. Facade [позволяет вызывать методы других классов]
17. Fluent interface (Удобный интерфейс) [вызов методов через стрелочку как в laravel (return $this), обычно используется для построения SQL запросов]
18. Flyweight (Приспособленец) [Используется, чтобы разгрузить память]
19. Proxy [чтобы не вызывать методы самого объекта, а proxy будет направлять к самому родительскому классу и родительский класс будет давать реализацию метода, который вызывается]


Behavioral (Поведенческие) [предназначены, чтобы повысить коммуникацию между классами и объектами]

20. State (Состояние) [Есть объект с разными статусами и шаблон позволяет менять статусы]
21. Strategy [Есть несколько стратегий реализации и в том или ином случае запускаем какую то из стратегий]
22. Object null
23. Command [есть интерфейс с методом выполнить и есть расширение этого интерфейса где есть метод запретить выполнение. объект на основе интерфейса с выполнить выполняется, а объект на основе интерфейса запретить выполнение не выполняется]
24. Interpreter [шаблон предназначен чтобы писать интерпретаторы языка, создавать свой язык программирования]
25. Specification [ученик получил какую то оценку и будут объекты на основе классов которые будут определять проходит он необходимый уровень или не проходит]
26. Chain [есть классы которые связаны с друг другом, но по одному условию. Если один класс не способен выполнить какую то поставленную задачу он делегирует это другому классу]
27. Iterator [Есть книга которую можно перелистывать и получать результат на необходимой какой-то странице]
28. Mediator [Есть некоторые классы между которыми появляется класс Mediator и снижает их связанность и позволяет более независимо работать приложению]
29. Memento (Хранитель) [Есть первый класс. Создаем класс второй класс, который хранит копию первого класса для получения нужной информации]
30. Observer (Наблюдатель) [Напоминает watch. Слежение за состоянием объекта. Если были какие-то изменения - этот объект сохраняется в массиве и мы может его оттуда получить]
31. Template method [Есть абстрактный родитель у которого есть реализация методов. Наследники могут переписать часть методов под себя. Но методы наследников можно вызвать только у родителя ]
32. Visitor [Есть класс, который может хранить в массиве разные объекты и дает возможность вызвать эти объекты и вызывать методы объектов из массива]
